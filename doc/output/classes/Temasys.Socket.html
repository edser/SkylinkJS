Parameters:

  • [options] JSON optional

    The options.

    • [server] @[exp] String optional

      The custom Signaling server domain to use.

    • [ports] @[exp] Array optional

      The custom list of Signaling server ports (Number) to use.

      • The priority of port used is based on first index order starting from 0.
    • [path] @[exp] String optional

      The custom Signaling server path to use.

    • [protocol] String optional

      The protocol to use to connect to the Signaling server.

      • When not provided, the current accessing window.location.protocol will be used.
    • [transports] Array optional

      The list of socket.io-client transports to use.

      • Available transports are: "websocket" (Websocket) and "polling" (Polling).
      • The priority of transport to use are based on first index order starting from 0.
      • When not provided, ("websocket", "polling") will be used.
      • If the browser does not support WebSocket API, "websocket" transports will be ignored.
    • [compressData=false] Boolean optional

      The flag if data sent should be compressed.

    • options JSON

      The socket.io-client options configured for each socket.io-client transport type.

      • The current default is { websocket: { reconnection: true, reconnectionAttempts: 2, reconnectionDelay: 5000, reconnectionDelayMax: 2000, randomizationFactor: 0.5, timeout: 20000 }, polling: { reconnection: true, reconnectionAttempts: 4, reconnectionDelay: 2000, reconnectionDelayMax: 1000, randomizationFactor: 0.5, timeout: 20000 } }
      • index JSON
        @[exp] The socket options for the "index" socket.io-client transport type. - "index" can be identified as: "websocket" (Websocket) or "polling" (Polling).
      • [index.reconnection=true] Boolean optional
        The flag if socket connection should reconnect several attempts for the current transport or port used before switching to the next available transport or port available.
      • [index.reconnectionAttempts] Number optional
        The reconnection attempts to take if options.options.index.reconnection is enabled. - The maximum value that can be provided is 5.
      • [index.reconnectionDelay] Number optional
        The number of miliseconds to wait before starting the next reconnection attempt, which is affected by the randomizationFactor configured.
      • [index.reconnectionDelayMax] Number optional
        The maximum number of miliseconds to wait before starting the next reconnection attempt.
      • [index.randomizationFactor] Number optional
        The randomization for each reconnection attempt. - The range is from 0 to 1.
      • [index.timeout] Number optional
        The timeout in miliseconds to consider that the inital connection has timed out.
