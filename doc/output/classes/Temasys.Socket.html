Temasys.Socket Class

Defined in: source/components/socket.js:1

Available since 0.7.0

Handles the socket connection to the Signaling server. This class is only to be initialised from Room class.

)

Defined in source/components/socket.js:1

Available since 0.7.0

Parameters:

  • [options] JSON optional

    The options.

    • [server] String optional

      The Signaling server domain to connect to. The default is based on the server domain provided by the Auth (API) server.

    • [ports] Array optional

      The list of Signaling server ports to connect to. Priority of port to use are based on first index order starting from 0. The default is based on the list of ports provided by the Auth (API) server.

      • #index Number
        The port number to use.
    • [path] String optional

      The Signaling server path to connect to. The default is /socket.io.

    • [protocol] String optional

      The Signaling server protocol to connect to. The default is the value of the current accessing window.location.protocol.

    • [transports] Array optional

      The list of socket transports to use. Priority of transport to use are based on first index order starting from 0. The default is ("websocket", "polling"), or ("polling") only if Websocket transport is not supported.

      • #index String
        The transport item to use. Available options are: "websocket" (Websocket) and "polling" (Polling).
    • [compressData] Boolean optional

      The flag if data sent to Signaling server should be compressed. The default is false.

    • options JSON

      The socket options for each transport type.

      • #transport JSON
        The socket options for transport type. The #transport key is either "websocket" (Websocket) or "polling" (Polling).
      • [#transport.reconnection] Boolean optional
        The flag if socket should attempt reconnection for each port and transport type. The default is true.
      • [#transport.reconnection] Number optional
        The flag if socket should attempt at least several reconnections for current port and transport type before switching to the next available port or transport. The defaults are true for both Websocket and Polling.
      • [#transport.reconnectionAttempts] Number optional
        The reconnection attempts to take if reconnection is enabled for each port and transport type. The defaults are 2 for Websocket and 4 for Polling. The maximum is 5.
      • [#transport.reconnectionDelay] Number optional
        The number of miliseconds to initially wait before attempting a new reconnection. This is affected by the randomization factor range. The defaults are 5000 for Websocket and 2000 for Polling.
      • [#transport.reconnectionDelayMax] Number optional
        The maximum number of miliseconds to wait between reconnections. This increases the reconnection delay for randomization factor range before waiting to start a reconnection. The defaults are 2000 for Websocket and 1000 for Polling.
      • [#transport.randomizationFactor] Number optional
        The randomization factor range from 0 to 1. The defaults are 0.5 for both Websocket and Polling.
      • [#transport.timeout] Number optional
        The timeout in miliseconds to consider that the connection timed out. The defaults are 20000 for both Websocket and Polling.
