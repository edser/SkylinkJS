      The STUN ICE servers settings.

      The flag if public STUN ICE servers should be used.

    • [constraints.iceServers.stun.urls] Array | String optional

      The list of STUN servers urls to use. E.g. ("turn.temasys.io:3478", "turn.temasys.io:19305") or "turn.temasys.io"

    • [candidates.relay=true] Boolean optional

      The flag if "relay" ICE candidates should be used.

      The codecs settings.

      • This configures the Peer connection native RTCSessionDescription object string.
    • [codecs.params] JSON optional

      The media connection codecs parameters settings.

      • When settings are not provided, the default browser configuration is used.
    • [codecs.params.audio] JSON optional

      The media connection audio codecs settings.

    • [codecs.params.audio.opus] JSON | String optional

      The media connection OPUS audio codec settings.

      • When provided as a String, it sets as the full parameter settings. E.g. minptime=1;stereo=1
    • [codecs.params.audio.opus.stereo] Boolean optional

      The flag if OPUS audio codec stereo band should be configured for sending audio packets.

    • [codecs.params.audio.opus.usedtx] Boolean optional

      The flag if OPUS audio codec should enable DTX (Discontinuous Transmission) for sending audio packets. This might help to reduce bandwidth as it reduces the bitrate during silence or background noise, and goes hand-in-hand with the options.media.audio.voiceActivityDetection flag.

    • [codecs.params.audio.opus.useinbandfec] Boolean optional

      The flag if OPUS audio codec has the capability to take advantage of the in-band FEC (Forward Error Correction) when sending audio packets. This helps to reduce the harm of packet loss by encoding information about the previous packet loss.

    • [codecs.params.audio.opus.maxplaybackrate] Number optional

      The OPUS audio codec maximum output sampling rate in Hz (hertz) that is is capable of receiving audio packets, to adjust to the hardware limitations and ensure that any sending audio packets would not encode at a higher sampling rate specified by this.

      • This value must be between 8000 to 48000.
    • [codecs.params.audio.opus.minptime] Number optional

      The OPUS audio codec encoder minimum length of time in milleseconds should be encapsulated for a single audio data packet.

    • [codecs.params.audio.telephoneEvent] JSON | String optional

      The media connection DTMF audio codec settings.

      • When provided as a String, it sets as the full parameter settings. E.g. 0-5
    • [codecs.params.audio.pcma] JSON | String optional

      The media connection PCMA (G711a) audio codec settings.

      • When provided as a String, it sets as the full parameter settings.
    • [codecs.params.audio.pcmu] JSON | String optional

      The media connection PCMU (G711u) audio codec settings.

      • When provided as a String, it sets as the full parameter settings.
    • [codecs.params.audio.g722] JSON | String optional

      The media connection G722 audio codec settings.

      • When provided as a String, it sets as the full parameter settings.
    • [codecs.params.audio.isac] JSON | String optional

      The media connection ISAC audio codec settings.

      • When provided as a String, it sets as the full parameter settings.
    • [codecs.params.audio.ilbc] JSON | String optional

      The media connection iLBC audio codec settings.

      • When provided as a String, it sets as the full parameter settings.
    • [codecs.params.video] JSON optional

      The media connection video codecs settings.

    • [codecs.params.video.h264] JSON | String optional

      The media connection H264 video codec settings.

      • When provided as a String, it sets as the full parameter settings. E.g. levelAsymmetryAllowed=1;packetizationMode=1
    • [codecs.params.video.h264.profileLevelId] String optional

      The H264 video codec base16 encoded string which indicates the H264 baseline, main, or the extended profiles.

    • [codecs.params.video.h264.levelAsymmetryAllowed] Boolean optional

      The flag if H264 video codec encoder of video packets should be at a different level from decoder of video packets.

    • [codecs.params.video.h264.packetizationMode] Boolean optional

      The flag if H264 video codec packetization mode should be enabled where it splits video frames that are larger for a RTP packet into RTP packet chunks.

    • [codecs.params.video.vp8] JSON | String optional

      The media connection VP8 video codec settings.

      • When provided as a String, it sets as the full parameter settings.
    • [codecs.params.video.vp8.maxFr] Number optional

      The maximum number of frames per second (fps) the VP8 video decoder is capable of decoding video packets.

    • [codecs.params.video.vp8.maxFs] Number optional

      The maximum number of frame size macroblocks that the VP8 video decoder is capable of decoding video packets.

      • The value has to have the width and height of the frame in macroblocks less than the value of parseInt(Math.sqrt(maxFs * 8)). For example, if the value is 1200, it is capable of the support of 640x480 frame width and height, which heights up to 1552px (97 macroblocks value.
    • [codecs.params.video.vp9] JSON | String optional

      The media connection VP9 video codec settings.

      • When provided as a String, it sets as the full parameter settings.
    • [codecs.params.video.vp9.maxFr] Number optional

      The maximum number of frames per second (fps) the VP9 video decoder is capable of decoding video packets.

    • [codecs.params.video.vp9.maxFs] Number optional

      The maximum number of frame size macroblocks that the VP9 video decoder is capable of decoding video packets.

      • The value has to have the width and height of the frame in macroblocks less than the value of parseInt(Math.sqrt(maxFs * 8)). For example, if the value is 1200, it is capable of the support of 640x480 frame width and height, which heights up to 1552px (97 macroblocks value.
    • [codecs.prefer] JSON optional

    • [codecs.prefer.audio.codec.channels] Number optional

    • [codecs.prefer.video.codec.samplingRate] Number optional

      The preferred video codec sampling rate if available for the preferred video codec name.

    • [codecs.mechanism] JSON optional

      The codecs RTCP or FECs settings.

      The flag to enable ULPFEC codec (generic forward error correction) mechanism when available.

    • [codecs.mechanism.red=true] Boolean optional

      The flag to enable RED codec (generic forward error correction) mechanism when available.

    • [codecs.mechanism.rtx=true] Boolean optional

      The flag to enable RTX (re-transmission) mechanism when available.

    • [codecs.mechanism.remb=true] Boolean optional

      The flag to enable REMB (receiver estimated maximum bitrate) mechanism when available.

    • [codecs.mechanism.nack=true] Boolean optional

      The flag to enable NACK (negative acknowledgement) mechanism when available.

    • [codecs.mechanism.ccm=true] Boolean optional

      The flag to enable CCM (codec control messages) mechanism when available.

    • [codecs.mechanism.transportcc=true] Boolean optional

      The flag to enable transport-cc (transport-wide congestion control) mechanism when available.

    • [media] JSON optional

      The Peer connection media connection settings.

      • This configures the Peer connection native RTCSessionDescription object string.
      • [audio] JSON | Boolean optional
        The media audio streaming settings. - When provided as false, this disables the audio streaming connection entirely.
      • [audio.send=true] Boolean optional
        The flag if audio packets should be sent to Peer connection. - When provided as false, this prevents sending audio packets to Peer despite audio tracks being added.
      • [audio.receive=true] Boolean optional

      • [audio.voiceActivityDetection=true] Boolean optional
        The flag if voice activity detection (VAD) should be enabled.
      • [audio.dtmf=false] Boolean optional
        The flag if DTMF should be created if supported.
      • [audio.maxBandwidth=20] Number optional
        The maximum range limit of sending audio bandwidth in kbps. - The bandwidth might be lower or higher by a few kbps than the specified range at times. - Set it as 0 to remove limits.
      • [video] JSON | Boolean optional
        The media video streaming settings. - When provided as false, this disables the video streaming connection entirely.
      • [video.send=true] Boolean optional
