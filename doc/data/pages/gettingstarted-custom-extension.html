Screensharing extensions are required to enable Screensharing functionalities for Web App domains, in which is only available on Firefox, Chrome and Opera (depending on Temasys AdapterJS version used) for Mac, PC and Ubuntu devices. For IE and Safari (which uses the Temasys WebRTC plugin) browsers, commerical licensing would be required to enable Screensharing functionalities.

By default, the Temasys AdapterJS uses the default Temasys Web SDK extensions which is enabled for the Temasys Web SDK demo domains (e.g. getaroom.io) and the localhost server (so it works on localhost domain). To generate and integrate your own extensions, you may follow the steps below, and skip any steps when necessary.

1. Generating and Downloading the Screensharing extensions

For Firefox browsers, you may access the Developer Console and generate the Screensharing extension under the App Key settings "Extension" tab. Do note that by default, it takes into account of the CORS domains configured for the App Key and sets it as the domains to enable Screensharing functionalities.

For Chrome or Opera browsers, you might have to contact the Support Portal to obtain the Screensharing extension files.
